# Module 3 Introduction: Data Cleaning and Project Kick-Off

Last week, you acted like a data detective by inspecting the dataset and identifying possible quality issues. This week, you begin the first careful cleaning pass. The goal is not to make a perfect final dataset immediately. The goal is to make controlled, documented improvements while preserving the meaning of the data.

Cleaning decisions matter. Converting a column, removing duplicates, dropping rows, or filling missing values can change the story the data tells. In this module, you will practice basic Pandas cleaning operations and begin connecting those habits to your individual semester project.

## This Week You Will

- identify common cleaning issues such as missing values, duplicates, incorrect types, and inconsistent categories;
- create a working copy of a raw dataset before cleaning;
- convert numeric and date fields using Pandas;
- inspect full-row and key-based duplicates;
- consider basic missing-data strategies;
- document cleaning decisions in a clear, traceable way.

## Lab Focus

The lab uses an intentionally messy employee dataset. You will load the data, create a clean working copy, correct data types, inspect duplicates, review missing values, verify the result, and maintain a basic cleaning log.

## Project Connection

This module also kicks off the Data-to-Insight Challenge, your semester-long individual project. Milestone 1 asks for an initial data profile and basic cleaning plan. You do not need to fully clean your project data yet, but you should begin identifying specific data quality issues and writing a plan for how you might address them.

## Key Habit

Preserve the raw data, work on a copy, verify after changes, and explain what you did. Cleaning is not just code. It is a set of decisions another analyst should be able to understand.
