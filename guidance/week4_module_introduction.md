# Module 4 Introduction: GitHub, Reproducibility, Governance, and Ethics

This week is a practical pause before we move deeper into transformation and integration. It is intentionally lighter on new coding, but it is not filler. The purpose is to make your technical work more professional, reproducible, and responsible.

Cleaning a dataset once is not enough. In an analytics project, someone should be able to understand what changed, why it changed, whether the work can be rerun, and whether the data was handled responsibly. This module connects GitHub workflow, notebook organization, clean Python style, data governance, privacy, ethics, and responsible tool use.

## This Week You Will

- use GitHub commits and pushes as an evidence trail;
- organize project folders and notebooks more clearly;
- use reusable paths and clearer Python variable names;
- add markdown explanations to support reproducibility;
- draft a simple data dictionary;
- identify PII, quasi-identifiers, and sensitive fields;
- reflect on governance, ethics, and responsible AI/tool use.

## Lab Focus

The lab asks you to improve notebook readability, revise unclear variable names, write observations about missing values, improve data dictionary rows, identify privacy risks, create a review copy without direct names, and commit your work with a meaningful message.

## Project Connection

Your semester project will become harder if documentation waits until the end. Use this week to build the professional floor under the rest of the project: organized files, readable notebooks, meaningful commits, and short notes explaining data handling decisions.

## Key Habit

Your repository is not only where you store work. It is where you show judgment.
