## What You Will Practice

You will practice reproducible file paths, descriptive variable names, notebook organization, short analytical observations, basic data dictionary drafting, identifying PII and sensitive fields, creating a processed output, and checking your GitHub workflow.

## Recommended Workflow

1. Open the notebook in Codespaces or local Jupyter.
2. Run the import and path setup cells.
3. Load the data with a descriptive variable name.
4. Complete the code style and notebook organization sections.
5. Draft or improve the data dictionary.
6. Identify fields that may create privacy or governance concerns.
7. Create the processed output.
8. Complete the GitHub workflow checkpoint.
9. Save your work, then commit and push if using GitHub.

## What to Pay Attention To

Reproducibility means someone else can understand and rerun your work. Governance means you can explain what the data contains, what might be sensitive, and how it should be handled. Both are part of professional data work.
